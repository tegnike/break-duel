## 0. 制約（ユーザー指定・2026-07-09）

- **カード枚数（カードプール）は変更しない。新カードの追加はしない**
- 対策は**既存カードの再調整**（効果・数値・条件の変更）で行う
- デッキ構成の入替（どのカードを何枚入れるか）は補助的に可。ただし第四弾 B1〜B7 で
  「汎用回答の投入はシーソーを起こす」と実証済みのため、**入替だけで解こうとしない**

## 1. 目標と非目標

**目標**:

- p2-3 ストレスの **break/control 合算を 50% 以下**へ（監視解除水準）。
  総合値も 60% 警報線から明確に引き離す
- p3（fair-gen006 で総合 51.85% / break+control 52.60%）も同時に 50% 以下へ戻す
- fair-gen006 基準の6デッキリーグを単色45-55% / 先攻48-52%へ戻す

**非目標（壊してはいけないもの）**:

- 6 デッキリーグの帯（単色 45-55% / 先攻 48-52%）
- 盛り上がり指標（draw ≤2% / 20-30T / リード交代 50%+ / 逆転 30-45% / スノーボール ≤70%）
- beginner 較正 5-20% / tutorial / `npm run check`

## 2. 設計方針 — 「条件付きアンチスワーム」（第四弾の教訓の裏返し）

第四弾のシーソー: 無条件に強い回答（PURGE 系）を break/control に足すと、p2-3 は下がるが
**対 6 デッキ戦でも強くなりすぎて**リーグ帯が壊れた。

したがって本命は、**スワーム（横に並べる相手）にだけ効き、通常戦ではほぼ現状どおり**の
条件付き効果への既存カード改修。条件の設計素材（仮説。最終判断は B0 診断に従う）:

- 「**相手の場に召喚獣が 3 体以上**のとき」効果が強化される（対スワーム限定の火力/除去/防御）
- 「**power 3 以下の攻撃**に対して」防御値ボーナス（スワームの主武装だけを受ける盾）
- 「相手の場の召喚獣の数」に比例する効果（幅を取ることをリスクに変える）

改修先の選定原則:

- **使用率の低い既存カード**（`card_usage` 統計で特定）を優先——「死にカードの救済」と
  「スワーム回答の供給」を 1 変更で兼ねる
- **break / control のデッキに既に入っているカード**を優先——デッキ入替なしで回答が配備され、
  シーソーの再発リスクが最小
- 単色デッキのアイデンティティ（fire=速攻 / water=ドロー / wind=テンポ / earth=持久）を壊さない
- 却下済み案に抵触しない（無条件除去・コスト割引系は禁止）

## 3. 進め方

**B0 — 診断（カードを触る前に。コード変更なし）**

1. 最新チャンピオンでの p2-3 ストレスを再計測し、勝ち筋を分解する
   （どの盤面幅・どの power 帯の攻撃で勝つか。第三弾診断「power3 攻撃が power1-2 防御を
   踏み抜く」が新世界でも主経路かを確認）
2. fair-gen006 リーグの water 70.5% / fire 40.9% / wind 44.0% / 先攻45.8% をペア別・カード別に分解し、
   water の過剰な勝ち筋と fire/wind の損失経路を特定する
3. `card_usage` 統計から**使用率の低い既存カード**を列挙し、break/control 採用カードと
   突き合わせて改修候補 3〜5 枚を選ぶ
4. 各候補に改修案（条件付き効果）と**事前予測**（p2-3 / リーグ / 盛り上がりがどう動くか）を書く

**B1 — 改修実験（1 カード 1 変更 = 1 検証）**

- 各改修: 実装 + `cardEffectCoverage.test.ts` 登録 + カードテキスト更新 →
  **p2-3 ストレス（1000/order 推奨、最低 500）+ リーグ 2 シード**で計測
- 効果があった改修は残し、次の候補を重ねる（累積で 50% 以下を目指す）
- 逆効果・無風の改修は戻して記録（再提案防止）

**B2 — 収束確認（全数値の一括再計測）**

- full ストレス（p2-3 / p3 / 全候補）+ リーグ 2 シード + 盛り上がり 1000 戦 +
  beginner 較正 + tutorial
- §1 の目標と非目標をすべて満たしたら合格

**B3 — 記録**

- `docs/game-spec.md` / カードテキスト / UI 文言の同期（card-addition スキルの
  file-map を参照。**アートは既存カードの流用なので変更不要**）
- `test:balance` の p2-3 閾値を戻すか判断（50% 以下で安定したら警報線 0.60 → 従来の
  0.50 ゲートへ復帰させ、監視項目を解除）
- balance-history 記録 + design-principles の「スワーム回答」方針の達成状況を更新

## 4. やらないこと

- 新カードの追加・カードプール枚数の変更（ユーザー指定の制約）
- コアルールの変更（終盤設計は確定済み。廃案リストの再提案禁止）
- CPU の変更（最新世代を凍結。挙動がおかしければ記録して CPU 計画へ分離）
- 無条件除去など却下済み案の再提案

## 5. リスクと撤退線

- 条件付き効果は CPU が使いこなせない可能性がある（評価関数が条件を見ない）。
  その場合「CPU の最小評価対応」が必要になるが、カード変更と別コミットで行い、
  ガントレット非退行を確認する
- 改修 5 枚まで試して合算 50% 以下に届かない場合は、現状の到達値と残る選択肢
  （警報線の恒久化 / 将来のカード追加時に回答を設計）をまとめて**ユーザー判断待ちで終了**

## 6. （参考）本計画の位置づけ

これは「ゲームを壊さずに、縮退ベンチマークへの余裕を取り戻す」品質改善タームであり、
失敗してもゲームは現状（全ゲート合格 + 監視 1 件）のまま。焦らず 1 カードずつでよい。
