# Break Duel ゲーム仕様

最終更新: 2026-06-23

この文書は、現在の `Break Duel` 実装の正仕様です。ブラウザ UI と Python シミュレータはこの仕様に従います。実装互換のため、コード内の type 値やカード ID には `ai` / `AI-*` / `MEM-*` などの旧キーが残っています。

## 1. ゲーム概要

`Break Duel` は 2 人対戦の小型 TCG です。各プレイヤーは 20 枚デッキを使い、キャラクターカードを場に出して攻撃し、相手のライフを 0 にすることを目指します。

現在のカード種別は次の 3 種類です。

| 種別 | 実装上の type | 役割 |
| --- | --- | --- |
| 召喚獣 | `ai` | 場に出して攻撃・防御するキャラクターカード |
| 指令 | `event` | 1 回使い切りのアクションカード |
| 遺物 | `memory` | 1 枚だけ配置できる継続効果カード |

## 2. 標準設定

| 項目 | 値 |
| --- | ---: |
| 初期ライフ | 5 |
| デッキ枚数 | 20 |
| 先攻初期手札 | 5 |
| 後攻初期手札 | 4 |
| 通常アクション数 | 2 |
| 先攻 1 ターン目アクション数 | 1 |
| 場の召喚獣上限 | 3 |
| 最大手番数 | 60 |
| 属性相性補正 | なし |
|召喚獣個別効果の補正 | カードごと |
| power 3/4 通常登場コスト | 2 |
| 手札防御上限 | 1 回 / ターン |
| 手札上限 | なし |

標準設定の例外:

- 先攻の 1 ターン目はターン開始ドローなし。
- 先攻の 1 ターン目は攻撃できない。
- 後攻の 1 ターン目は通常通り 1 枚引き、2 アクションで行動する。
- power 4 は攻撃後に力を使い切って退場してトラッシュへ行く。
- 消耗中の召喚獣は攻撃も防御もできない。

## 3. プレイヤー

標準対戦では 2 人のプレイヤーを使います。

| index | UI 表示名 | デッキ |
| ---: | --- | --- |
| 0 | あなた | 紅蓮突破デッキ |
| 1 | ライバル | 大地守護デッキ |

先攻は常に `あなた` です。初期シードの標準値は `1` です。デッキはシード付き乱数でシャッフルされます。

## 4. 勝敗

勝敗判定は次の順で発生します。

1. ライフが 0 以下になったプレイヤーがいれば、その相手が勝利する。
2. 両者の山札・手札・場の召喚獣がすべて空になった場合、残りライフが多い側が判定勝ち。同点なら引き分け。
3. 最大 60 手番に到達して勝者がいなければ、残りライフが多い側が判定勝ち。同点なら引き分け。

リソース枯渇判定では、トラッシュと遺物は数えません。トラッシュや遺物だけでは攻撃もドローも発生しないためです。

## 5. カードプール

### 5.1召喚獣カード

召喚獣カードは 4 属性 x 4 power x A/B 2 系統の 32 種です。既存互換のため、A 系統は従来通り `B` なしの ID を使います。

| power | 役割名 | 通常登場コスト | 基本挙動 |
| ---: | --- | ---: | --- |
| 1 | 小型 | 1 | 低コストのアップグレード元 |
| 2 | 中継 | 1 | 低コストの中継役 |
| 3 | 中型 | 2 | 中型召喚獣。個別効果を持つカードがある |
| 4 | 切札 | 2 | 攻撃後に力を使い切ってトラッシュへ行く |

カード ID は A 系統が `AI-{属性コード}-{power}`、B 系統が `AI-{属性コード}-{power}B` です。

| 属性 | 属性コード | カード ID |
| --- | --- | --- |
| 火 | `FIRE` | `AI-FIRE-1` から `AI-FIRE-4B` |
| 水 | `WATER` | `AI-WATER-1` から `AI-WATER-4B` |
| 風 | `WIND` | `AI-WIND-1` から `AI-WIND-4B` |
| 土 | `EARTH` | `AI-EARTH-1` から `AI-EARTH-4B` |

召喚獣カードの一部だけが個別効果を持ちます。すべての召喚獣が効果を持つわけではありません。

| カードID | 個別効果 |
| --- | --- |
| `AI-FIRE-1B` | 攻撃が防御された時、相手は手札を 1 枚捨てる |
| `AI-FIRE-2` | 攻撃値 +1 |
| `AI-FIRE-2B` | 手札防御されても相手に 1 ダメージ |
| `AI-FIRE-3B` | 攻撃値 +1。ただし手札防御に使えない |
| `AI-FIRE-4` | 攻撃後に退場した時、1 枚引く |
| `AI-FIRE-4B` | 相手ライフが 2 以下なら手札防御されない。ただし登場時、自分に 1 ダメージ |
| `AI-WATER-1` | 登場時に 1 枚引く |
| `AI-WATER-1B` | 登場時に 1 枚引く。ただし手札防御に使えない |
| `AI-WATER-2` | 登場時に 2 枚引き、手札を 1 枚捨てる |
| `AI-WATER-2B` | 攻撃が防御された時、1 枚引く。ただし手札防御に使えない |
| `AI-WATER-3` | 登場時に 1 枚引く |
| `AI-WATER-3B` | 登場時に 2 枚引き、手札を 1 枚捨てる |
| `AI-WATER-4B` | 攻撃後に退場した時、2 枚引く。ただし登場時、相手は 1 枚引く |
| `AI-WIND-1` | 攻撃しても消耗しない |
| `AI-WIND-1B` | 攻撃しても消耗しない |
| `AI-WIND-2B` | 登場時に相手の未消耗召喚獣1体を消耗させる |
| `AI-WIND-3` | 登場時に相手の未消耗召喚獣1体を消耗させる |
| `AI-WIND-3B` | 登場時に 1 枚引き、自分の消耗中召喚獣1体を未消耗に戻す |
| `AI-WIND-4B` | 攻撃後退場時、トラッシュではなく手札に戻る。ただし消耗で出て、手札防御に使えない |
| `AI-EARTH-1B` | 場防御成功時、1 枚引く |
| `AI-EARTH-2` | 防御値 +1 |
| `AI-EARTH-2B` | 防御値 +1 |
| `AI-EARTH-3B` | 登場時、手札が 1 枚以下ならトラッシュの召喚獣1枚を手札に戻す |
| `AI-EARTH-4` | 登場時、手札が 1 枚以下ならトラッシュの召喚獣1枚を手札に戻す |
| `AI-EARTH-4B` | 場防御成功時、1 枚引く。ただし消耗で出る |

現在のブラウザ UI では、各召喚獣カードに `src/assets/card-art/*.webp` の 16:9 イラストが割り当てられています。

### 5.2 指令カード

指令カードはすべて 1 アクションで使用し、使用後にトラッシュへ行きます。

| カードID | カード名 | 使用条件 | 効果 |
| --- | --- | --- | --- |
| `CMD-OPTIMIZE` | 戦術整理 | 自分の手札が 2 枚以上 | 手札を 1 から 2 枚捨て、2 枚引く |
| `CMD-PATCH` | 癒し薬草 | 自分の消耗中召喚獣がいる | 自分の消耗中召喚獣1体を未消耗に戻す |
| `CMD-DISRUPT` | 絡め蔦 | 相手の未消耗召喚獣がいる | 相手の未消耗召喚獣1体を消耗させる |
| `CMD-RELEARN` | 追憶の巻物 | 自分のトラッシュに召喚獣がある | トラッシュの召喚獣1枚を手札に戻す。手札が残っていれば 1 枚捨てる |
| `CMD-SANDBOX` | 守護結界 | 残り 2 アクション以上、攻撃可能、未消耗 power 4 がいる、守護結界効果未使用 | このターン、次の power 4 攻撃後退場を 1 回防ぐ |

自動選択の対象:

- `癒し薬草`: 自分の消耗中召喚獣のうち power が最も高いカード。同 power なら ID の降順。
- `絡め蔦`: 相手の未消耗召喚獣のうち power が最も高いカード。同 power なら ID の降順。ブラウザ UI で人間が使う場合は対象を手動選択する。
- `追憶の巻物`: 自分のトラッシュにある召喚獣のうち power が最も高いカード。同 power なら ID の降順。
- コストや捨て札を自動で選ぶ場合は、低優先度カードから捨てる。優先度は召喚獣の power、指令/遺物は 1 相当。同優先度なら ID 昇順。

### 5.3 遺物カード

遺物カードは 1 アクションで配置します。各プレイヤーは遺物を 1 枚だけ保持できます。既存遺物がある状態で新しい遺物を配置した場合、既存遺物はトラッシュへ行きます。

| カードID | カード名 | 効果 |
| --- | --- | --- |
| `MEM-FIREWALL` | 守護の紋章 | 自分の場の召喚獣が同属性攻撃を場防御する時、手札を 1 枚捨てて防御値 +1 |
| `MEM-CACHE` | 旅人の鞄 | 自分のターン開始時、手札が 2 枚以下なら 1 枚引く |
| `MEM-PIPELINE` | 精霊の水脈 | 1 ターンに 1 回、自分が power 1 を登場させた時に追加で 1 枚引き、手札があれば 1 枚捨てる |

ブラウザ UI では、人間プレイヤーが `守護の紋章` と `精霊の水脈` の捨て札を手動選択します。自動側は低優先度カードから自動で捨てます。

## 7. 属性

属性は 4 種類です。

| 属性 | 表示色 | 得意傾向 |
| --- | --- | --- |
| 火 | 赤 | 防御されても手札やライフへ圧を残すが、自傷や防御不可の代償を持つ |
| 水 | 青 | ドロー、手札交換、防御された攻撃からの補充。ただし一部は相手にも補充を与える |
| 風 | 緑 | 消耗制御、相手妨害、攻撃後の手札帰還。ただし再利用札は消耗で出る |
| 土 | 茶 | 防御値、場防御成功時の補充、トラッシュ回収。ただし一部の守備札は遅い |

属性相性はありません。属性は「防御できるかどうか」を直接制限せず、どの属性でも防御候補になれます。属性ごとの差は、カード個別効果として表現します。

## 8. 防御判定

攻撃側召喚獣の power に個別効果を足した値を `攻撃値` とします。防御側は次の式で `防御値` を作ります。

```text
攻撃値 = 攻撃召喚獣のpower + 攻撃側召喚獣の個別効果
防御値 = 防御召喚獣のpower + 防御側召喚獣の個別効果 + 防御時ボーナス
```

防御時ボーナス:

- `守護の紋章` があり、場防御召喚獣と攻撃召喚獣が同属性で、手札を 1 枚捨てられるなら +1。

防御可能条件:

```text
防御値 >= 攻撃値
```

防御結果:

| 防御方法 | 比較 | 結果 |
| --- | --- | --- |
| 場防御 | 防御値 = 攻撃値 | 相打ち。攻撃召喚獣と防御召喚獣が両方トラッシュ |
| 場防御 | 防御値 > 攻撃値 | 防御成功。攻撃召喚獣はトラッシュ、防御召喚獣は場に残って消耗 |
| 手札防御 | 防御値 >= 攻撃値 | 攻撃を止める。手札防御召喚獣はトラッシュ、攻撃召喚獣は場に残る |
| 防御しない / 防御不能 | - | 防御側が 1 ダメージ |

防御成功時にダメージは発生しません。防御側はカードを引きません。

## 9. ターン進行

ターン開始時に次の処理を行います。

1. 手番数を 1 増やす。
2. そのターンのアクション数を設定する。
3. 両プレイヤーの手札防御使用回数を 0 に戻す。
4. ターンプレイヤーの場の召喚獣をすべて未消耗に戻す。
5. ターンプレイヤーの `精霊の水脈` 使用済みフラグをリセットする。
6. ターンプレイヤーの `守護結界` 保護をリセットする。
7. ターンプレイヤーの開始ターン数を 1 増やす。
8. ターン開始ドロー条件を満たすなら 1 枚引く。
9. `旅人の鞄` のターン開始効果を処理する。
10. リソース枯渇判定を行う。
11. 選択中カードを解除する。

ターン開始ドロー:

- 通常はターン開始時に 1 枚引く。
- 先攻の 1 ターン目だけ、標準設定では引かない。
- 山札が空なら 0 枚引く。トラッシュは自動で山札へ戻らない。

アクション数:

- 先攻の 1 ターン目は 1 アクション。
- 各プレイヤーの最初のターンは 2 アクション。ただし上の先攻 1 ターン目設定が優先される。
- それ以外は 2 アクション。

残りアクションが 0 以下になり、攻撃解決待ちでなければ自動的にターン終了します。

## 10. 行動

### 10.1召喚獣を場に出す

- 手札の召喚獣1枚を自分の場に出す。
- 場が 3 体で埋まっている場合は出せない。
- power 1/2 は 1 アクション。
- power 3/4 は 2 アクション。
- `登場時に 1 枚引く` 個別効果を持つ召喚獣は、登場時に 1 枚引く。
- `精霊の水脈` が自分の遺物にあり、そのターンに未使用なら、power 1 登場時に追加で 1 枚引き、手札があれば 1 枚捨てる。

### 10.2 アップグレード登場

- 手札の召喚獣は、場の同属性かつ低 power の召喚獣1体を元にアップグレード登場できる。
- 元カードはトラッシュへ行く。
- 新しい召喚獣は元カードがいた場スロットに入る。
- アップグレードコストは通常登場コスト -1、最低 1 アクション。
- 場が 3 体で埋まっていても、元カードと入れ替わるため実行できる。
- 登場時効果は通常通り発動する。

アップグレード元の自動選択:

- 同属性で power が低い候補から、power が最も低い場の召喚獣を選ぶ。
- 同 power なら ID 昇順。

### 10.3 遺物を配置する

- 手札の遺物 1 枚を遺物枠に置く。
- 既存遺物がある場合、既存遺物はトラッシュへ行く。
- 遺物配置は 1 アクション。

### 10.4 指令カードを使う

- 手札の指令カードを 1 枚使用する。
- すべて 1 アクション。
- 使用条件を満たさない指令カードは使用できない。
- 使用した指令カードはトラッシュへ行く。

### 10.5 攻撃する

- 自分の場の未消耗召喚獣1体で攻撃する。
- 攻撃した召喚獣は消耗する。
- 先攻の 1 ターン目は攻撃できない。
- 出したばかりの召喚獣でも、未消耗なら攻撃できる。
- power 4 が攻撃後に場に残っていれば、攻撃後退場でトラッシュへ行く。
- `守護結界` の保護があれば、次の power 4 攻撃後退場を 1 回だけ防ぎ、その power 4 は場に残って消耗する。

### 10.6 交換する

- 手札を 1 枚トラッシュし、1 枚引く。
- 1 アクション。
- 山札が空なら 0 枚引く。

### 10.7 ターン終了

- 残りアクションを 0 にし、相手のターンへ移る。
- 標準設定では手札上限がないため、ターン終了時の自動手札破棄は発生しない。
- `守護結界` の未使用保護はターン終了時に消える。

## 11. 攻撃と防御の解決順

攻撃解決は次の順序で行います。

1. 攻撃側が攻撃召喚獣を選ぶ。
2. 攻撃ログを追加する。
3. 防御側が人間なら攻撃宣言モーダルを表示し、防御選択待ちに入る。
4. 攻撃召喚獣を消耗する。
5. `pendingAttack` に攻撃情報を保持する。
6. 防御側が自動プレイヤーなら自動で防御選択を行う。
7. 防御側が人間なら、場防御・手札防御・防御しない、から選ぶ。
8. 場防御では、必要に応じて `守護の紋章` の捨て札選択を挟む。
9. 防御結果を解決する。
10. power 4 の攻撃後退場を処理する。
11. 勝敗、リソース枯渇、最大ターンを確認する。
12. 勝敗未決なら攻撃側をアクティブに戻し、攻撃コストとして 1 アクションを消費する。

注意:

- 防御側の選択中は、攻撃側のターンが一時停止している。
- 防御後、攻撃側の残りアクションが 0 になれば自動でターンが移る。
- 手札防御では攻撃召喚獣は場に残るが、攻撃済みなので消耗している。
- 場防御で攻撃召喚獣がトラッシュへ行った場合、power 4 の攻撃後退場処理は対象が場に残っていないため発生しない。

## 12. 手札防御

標準設定では、各プレイヤーは 1 ターンに 1 回まで手札防御できます。

条件:

- 手札に防御値が攻撃値以上になる召喚獣がある。
- このターンにまだ手札防御をしていない。
- 標準設定では、自分の場に召喚獣がいても手札防御できる。

手札防御成功時:

- 手札防御召喚獣は手札からトラッシュへ行く。
- 攻撃召喚獣は場に残る。
- 攻撃召喚獣は攻撃済みなので消耗している。
- 攻撃召喚獣が power 4 なら、守護結界がない限り攻撃後退場でトラッシュへ行く。
- 防御側はダメージを受けない。

## 13. 消耗

消耗中の召喚獣は次の制限を受けます。

- 攻撃できない。
- 防御できない。

消耗は自分のターン開始時に解除されます。

登場直後の扱い:

- 召喚獣は未消耗で場に出る。
- 出したターンでも攻撃可能条件を満たせば攻撃できる。
- 攻撃せずにターンを渡せば次の相手ターンに防御できる。

## 14. トラッシュ

カードは次の場合にトラッシュへ行きます。

- 指令カードを使用した。
- 交換で手札を捨てた。
- 指令や遺物効果で手札を捨てた。
- 場防御で攻撃召喚獣が退場した。
- 場防御で相打ちになり、防御召喚獣も退場した。
- 手札防御で防御召喚獣を使い切った。
- power 4 が攻撃後に退場した。
- 遺物配置で既存遺物が置き換えられた。
- アップグレード元になった。
- 任意設定の手札上限で捨てられた。

山札が空の状態でカードを引く場合、そのドローは 0 枚になります。トラッシュは自動では山札に戻りません。

## 15. 自動プレイヤーの行動方針

自動プレイヤーは決定論的な簡易ロジックで行動します。

行動優先順位:

1. 場が空なら、手札で最も power が高く、現在のアクション数で出せる召喚獣を出す。
2. 攻撃可能で、未消耗 power 4 と `守護結界` があり、守護結界使用条件を満たすなら `守護結界` を使う。
3. ダメージが通る攻撃があるなら、その攻撃を行う。
4. 場に空きがあるなら、手札で最も power が高く、現在のアクション数で出せる召喚獣を出す。
5. 可能なら、同属性の低 power 召喚獣を元にアップグレードする。
6. 遺物が未配置なら、手札の遺物を配置する。
7. 使える指令カードがあれば使う。
8. 攻撃可能な召喚獣がいれば、最も power が高い召喚獣で攻撃する。
9. 手札があれば、最も低優先度の手札を交換する。
10. 何もできなければターン終了する。

指令カードの優先順位:

1. `絡め蔦`
2. `癒し薬草`
3. `追憶の巻物`
4. `守護結界`
5. `戦術整理`

防御優先順位:

- 場防御と手札防御の成功候補をまとめる。
- power が最も低い候補を選ぶ。
- 同 power なら場防御を優先する。
- さらに同条件なら ID 昇順。

召喚獣の手札捨て優先順位:

- 召喚獣は power が低いほど低優先度。
- 指令/遺物は優先度 1 として扱う。
- 同優先度なら ID 昇順。

## 16. ブラウザ UI 仕様

### 16.1 画面構成

ブラウザ UI は次の領域で構成されます。

- 相手ステータスバー: デッキ名、ライフ、手札枚数、山札枚数、トラッシュ。
- 戦場: 相手の遺物/場、自分の遺物/場。
- 自分ステータスバー: デッキ名、ライフ、山札枚数、トラッシュ。
- 手札列: 自分の手札。
- 操作ドック: 選択カードの大きな 16:9 イラスト、詳細、アクションボタン、演出設定、個別効果、ログ。

### 16.2 カード選択と詳細

- 手札または自分の場のカードを選択できる。
- 選択中カードは枠が強調される。
- 選択中カードの詳細パネルには、16:9 の大きなカードアート、カード名、属性、power、コスト、個別効果が表示される。
- 手札のカードアート枠は小さく、詳細パネルでイラストを大きく確認できる。

### 16.3 アクションボタン

操作ドックには次のボタンがあります。

- 場に出す / 使用 / 遺物配置
- アップグレード
- 攻撃
- 交換
- ターン終了

選択カードや残りアクション、場の状態に応じて使用不可になります。

### 16.4 防御選択 UI

相手が攻撃した場合:

- 攻撃宣言モーダルが表示される。
- 同じモーダル内に防御選択 UI が表示される。
- 防御候補には「場」または「手札」のラベルが付く。
- 各候補には属性、power、防御値の式、結果見込みが表示される。
- 防御しないボタンも表示される。
- 防御を選ぶと攻撃宣言モーダルを閉じ、戦闘結果モーダルへ進む。
- 攻撃宣言モーダルを閉じた場合でも、未解決の防御選択 UI は操作ドック側に残る。

### 16.5 トラッシュ閲覧

- 自分と相手のトラッシュ表示はクリックできる。
- クリックすると前面モーダルでトラッシュ内容を表示する。
- トラッシュ内のカードを選択すると、詳細パネルで内容を確認できる。

### 16.6 演出

現在の UI には次の演出があります。

- 手札から場へ出すカード移動アニメーション。
- CPU が手札からカードを出す仕草のアニメーション。
- アップグレード時のカード移動アニメーション。
- 遺物配置時のカード移動アニメーション。
- 指令、交換、戦闘、ダメージ、攻撃後退場のモーダル演出。
- 戦闘結果では、攻撃、防御、勝敗、トラッシュ行きがカード状態として表示される。

### 16.7 演出確認設定

操作ドックの「演出」設定で、モーダル演出の進み方を切り替えられます。

| 設定 | 挙動 |
| --- | --- |
| 手動確認 | モーダルは自動で消えない。確認ボタンで閉じる |
| 自動送り | 種別ごとの表示時間後に自動で閉じる |

デフォルトは手動確認です。

自動送り時の基準時間:

- 戦闘: 3200ms
- ダメージ: 2900ms
- 登場 / アップグレード: 2600ms
- その他: 2400ms

現在、登場・遺物配置・アップグレードはカード移動アニメーションを主に使うため、通常のモーダルキューには入れません。

## 18. 検証

標準検証コマンド:

```bash
npm run check
```

このコマンドは次を実行します。

1. TypeScript 型チェック: `tsc --noEmit`
2. Vite ビルド: `vite build`
3. Python ルールテスト: `python3 -m unittest`

現在のルールテストには、次の重要ケースが含まれます。

- 属性相性がない防御判定。
- 召喚獣個別効果による攻撃値、防御値、登場時ドロー、手札交換、相手消耗、回収、攻撃後非消耗。
- 場防御の防御側生存と相打ち。
- 手札防御は 1 ターン 1 回。
- power 4 の攻撃後退場。
- `守護結界` による power 4 攻撃後退場防止。
- 先攻 1 ターン目の 1 アクション、ドローなし、攻撃不可。
- `旅人の鞄`、`守護の紋章`、アップグレード、手札上限任意設定。

単色リーグの最新確認例:

```bash
python3 -m ai_break_duel.cli league --games-per-pair 1000 --seed 4701 --out tmp/feature_league_4701
```

この設定では、4 単色デッキの勝率が概ね 46% から 53% に収まっています。

## 19. 既知の命名上の注意

ゲーム名は `Break Duel` です。表示テーマは異世界ファンタジーに寄せていますが、保存済みデッキ、テスト、シミュレータ互換のため、実装上の type 値、カード ID、Python パッケージ名には旧キーが残っています。

- Python パッケージ名: `ai_break_duel`
- 召喚獣カードの type 値: `ai`
- 召喚獣カード ID: `AI-FIRE-1` など
- 遺物カード ID: `MEM-FIREWALL` など
