## 0. 背景 — 第一弾で分かったこと

- 時計（C0 ライフ判定 + 手札上限、P1 衰弱）で膠着は完治する（全構成 draw ≤1.2%）→ **確定枠**
- 「見えない後出し手札防御」が病巣本体（P4a だけが逆転・リード交代・water 健全性を回復）
- ただし全廃は過剰: 手札防御は**アグロへの安全弁**も兼ねていた（全廃で p2-3 ストレスが RISK 化、
  試合短期化、スノーボール悪化）
- 結論: **「タダで見えない防御」と「防御ゼロ」の間**に正解がある。防御を「コストを払って持つもの」にする

## 1. 確定枠（全構成に同梱）

- **C0**: ターン上限 40 でライフ判定（同値のみ draw）+ 自ターン終了時の手札上限（既定 6。ノブ 7）
- **P1**: 自ターン開始時にドローできなければ 1 ダメージ（固定）
- 廃案の再提案禁止: P3（シージ）、P4a そのまま採用

## 2. 候補定義

### P4c — 制限つき手札防御（軽量・先行実験）

- 手札防御に使えるカードを **power 2 以下のみ**に制限（`handDefenseLimit: 1` は維持）
- 狙い: 大型攻撃は手札で止まらない（攻める理由が生まれる）が、小回りの防御と
  アグロへの安全弁は残る。ブラフも残る
- 実装: 手札防御の合法判定（`legalHandDefenders`）に power 条件を追加する程度の軽さ
- 前例: beginner は既にほぼこの仕様で動いている（水以外 power 2 以下）
- ノブ: 効きが弱ければ power 1 以下版（P4c-1）も試してよい

### P4b — セット防御（本命・情報構造の本改訂）

- 手札防御を廃止し、**「セット」アクション**を新設: 1 アクションを消費して手札のカード 1 枚を
  裏向きでセットスロット（1 枚まで）に置く。攻撃をブロックできるのは**セット札のみ**。
  ブロック処理は現行の手札防御と同等（使用後トラッシュ）
- 設計詳細（実装時の既定。変えたら記録）:
  - セット札の**存在**は公開情報、**中身**は非公開（→ ガードテストの対象に追加する。
    「相手のセット札の中身が違っても行動が同一」を新規テストで保証）
  - セットし直しは可（新しくセットすると旧セット札はトラッシュ。遺物と同じ扱い）
  - 遺物（memory タイプ）はセット不可（現行の手札防御不可と整合）
- 狙い: 防御にテンポコスト（1 アクション）がつき、ブラフが「伏せる」という可視の駆け引きになる。
  攻撃側はセット札の有無を見てリスク計算できる

### ノブ — P2a（ブレイクレート event 化）の再検証

- 第一弾では「誰も攻撃しない世界」だったため無風だった。攻撃が発生する P4b/P4c の世界で
  `drawOnAttackDamage: "point" → "event"` の on/off を各候補に掛け合わせて計測する

## 3. 実験マトリクスと判定基準

**実行順**（軽い順）: P4c → P4c+P2a → P4b → P4b+P2a →（必要に応じ P4c-1、手札上限 h7 ノブ）

各構成: リーグ 2 シード（4101 / 730001）+ 盛り上がり（break vs control 1000 戦）。
有望構成のみ full ストレス回帰（`games-per-order 80` 以上、全 7 候補完走）。

**一次判定（足切り）**:

| 指標 | 基準 |
| --- | --- |
| draw 率 | 2% 以下 |
| 平均ターン | **20〜30**（P4a の 18.1T は不合格だったことに注意） |
| ストレス回帰 | 全候補 OK（**特に p2-3**。break/control 合算でも 50% 以下） |

**二次判定（質の比較 + 対 P4a ベンチマーク）**:

| 指標 | 基準 | P4a h7 の実績（参考） |
| --- | --- | --- |
| リード交代あり | 50% 以上 | 55.4% |
| 2 点ビハインド逆転 | 35〜45% | 34.8% |
| 先に 2 点差をつけた側の勝率 | **65% 以下** | 72.5%（不合格だった） |
| 単色リーグ帯 | 45-55% に全デッキ近接 | 40.2-56.4% |
| water の健全性 | 帯内 | 45.9% |
| 先攻勝率 | 48-52% に近いこと（参考計測） | 45.8% |

**採用モデル**: 一次を全て満たし、二次で最良の 1 構成。同等なら変更範囲が小さい方（P4c 優先）。

## 4. CPU の扱い

- fair-gen005 の重み・探索は凍結。許可は**新ルールの最小評価対応のみ**:
  - P4c: 攻撃評価の手札防御推定に power 条件を反映（公開情報のみで計算可能）
  - P4b: 「セット」行動の評価と、攻撃評価での「相手セット札の有無」の考慮（存在は公開情報）。
    **相手セット札の中身は参照禁止**（新規ガードテストで機械的に保証）
- CPU が新ルールを明らかに下手に打っている兆候（例: 一度もセットしない）があれば記録し、
  採用判断を保留してユーザーに確認する
