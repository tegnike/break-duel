# v0.7.0 (2026-07-10)

終盤ルールを正式採用し、勝ち筋が見えた後の対戦がより締まるように調整したリリースです。
あわせて CPU の基準世代を更新し、今後のバランス調整を追いやすい形に整理しました。

## ハイライト

- **終盤ルールを正式採用**: 長引きすぎる展開を抑え、終盤の判断が勝敗につながりやすいルールへ更新しました。
- **CPU バランスを fair-gen006 基準へ更新**: 初心者向けCPUと通常CPUの挙動を再調整し、極端な一方勝ちや資源切れを抑えました。
- **防御と攻撃判断を調整**: 手札防御、場防御、リソース消費、攻撃上限まわりの判断を見直しました。
- **バランス検証記録を整理**: 完了済みの計画書をアーカイブし、現行の設計・履歴・追跡ドキュメントを読みやすくしました。

## 変更点

### バランス調整

- 終盤向けの追加ルールをゲーム本体とシミュレータに反映しました。
- fair-gen006 のCPUプロファイルと関連チャンピオンデータを追加しました。
- 初心者向け water CPU を終盤ルール後の基準に合わせて再調整しました。
- swarm 系への回答、防御コスト、攻撃上限、資源消費の挙動を検証結果に合わせて調整しました。

### 修正

- CPU の終盤ビーム探索で `end` 行動を候補から落とさないようにしました。
- セット防御ガードの条件を明示し、手札防御カウントの診断を追加しました。
- CodeRabbit 指摘に基づき、レビュー済みの実装細部を修正しました。

### その他

- CPU 改善用スキルと、AIガントレット・CPU診断・チューニング用スクリプトを追加しました。
- 完了済みの strongest CPU / fair CPU / endgame redesign 計画書を `docs/archive/` に整理しました。
- `docs/README.md`、`docs/balance-history.md`、`docs/game-spec.md`、設計原則、終盤設計レポートを更新しました。
- `package.json` / `package-lock.json` のバージョンを `0.7.0` に更新しました。
