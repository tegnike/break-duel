# 対戦キャラクター管理 実装計画

- 作成・実施日: 2026-07-10
- 状態: 実装・総合検証完了
- 対象: PCブラウザ

## 1. 目的

管理画面から対戦キャラクターを追加し、そのキャラクターに紐づくデッキ、CPU難度、複数イラスト、イベント別セリフ、音声を一元管理する。

対戦準備では相手設定を編集せず、登録済みキャラクターを選ぶだけにする。対戦自体は従来どおり1対1とする。

## 2. 最終仕様

### キャラクター管理

- `/admin/characters` にPC向け管理画面を置く。
- 組み込みキャラクターは読み取り専用で表示する。
- カスタムキャラクターは新規作成、編集、削除できる。
- 表示名と安定IDを登録する。
- 使用デッキはランダム、固定デッキ、保存済みデッキから選ぶ。
- CPU難度は初心者または挑戦者から選ぶ。
- 通常、被弾、喜び、切札、とどめの5画像を登録する。
- 15種類のゲームイベントごとに表示セリフと任意音声を登録する。
- 通常画像は必須、その他の画像は未登録時に通常画像へフォールバックする。
- 画像・音声はData URLへ変換し、IndexedDBへ保存する。

### 対戦準備

- 自分のデッキと対戦キャラクターだけを選択する。
- 相手デッキとCPU難度は選択キャラクターの設定を使用する。
- 相手の編集、複製、削除は対戦準備では行わない。
- 進行中の試合は開始時のキャラクターsnapshotを使用し、次戦向けの選択や編集の影響を受けない。

## 3. 非対象

- 3人以上が同時参加する対戦。
- 人間対人間、オンライン対戦、アカウント間同期。
- 管理者認証、サーバー保存、オブジェクトストレージ配信。
- CPU評価関数、難度バランス、カード、デッキルールの変更。
- スマホ向けレイアウトや狭幅用media query。
- 2人目の本番キャラクター素材制作。

## 4. データ設計

### 4.1 実行時キャラクター

`OpponentCharacterDefinition` を人物表現の共通境界とする。

- `id`
- `defaultDisplayName`
- `portraits`
- `lines`
- `deckSelection`
- `aiProfile`

`NIKE_CHARACTER` とIndexedDBから復元したカスタムキャラクターを、実行時カタログへ合流する。

### 4.2 カスタムキャラクター

`SavedOpponentCharacter` はversion付きで次を保持する。

- 表示名、ID、更新日時。
- デッキ選択、CPU難度。
- 5種類の画像Data URLと元ファイル名。
- 15 cueのセリフ、音声Data URL、元ファイル名。

保存先はIndexedDB `break-duel-character-admin` とする。容量の大きいData URLを`localStorage`へ保存しない。

### 4.3 互換プロフィールstore

`OpponentProfileStoreV1` は `break-duel:opponent-profiles` に次を保持する。

- `version`
- `selectedProfileId`
- `profiles[]`
  - `id`
  - `profileLabel`
  - `characterId`
  - `deckSelection`
  - `aiProfile`
  - `updatedAt`

このstoreは既存データとの互換およびキャラクター選択を維持する。対戦で使用するデッキ・CPU・表示素材の権威データは、選択されたキャラクター定義から解決する。読み込み時に不正項目を除外・選択修復した場合は、正規化後のstoreを保存し直す。

### 4.4 進行中snapshot

対戦開始時に、選択キャラクターの表示名、画像、セリフ、音声、デッキ、CPU難度を `ResolvedOpponentSnapshot` として確定する。

- 次戦のキャラクター選択は進行中snapshotを変更しない。
- キャラクターの編集・削除も進行中snapshotを変更しない。
- speech timer、Audioの`ended`、pending cueは`matchId`でガードする。

## 5. 実装境界

- `src/opponents/types.ts`: キャラクター、画像種別、voice cue、snapshotの型。
- `src/opponents/nike.ts`: 組み込みニケの素材・デッキ・CPU設定。
- `src/opponents/catalog.ts`: 組み込みとカスタムの実行時カタログ、fallback解決。
- `src/opponents/characterStorage.ts`: カスタムキャラクターの検証とIndexedDB CRUD。
- `src/opponents/storage.ts`: 互換プロフィールstoreのmigration、CRUD、永続化。
- `src/components/CharacterAdminPage.tsx`: キャラクター管理UI。
- `src/components/DuelSetupPanel.tsx`: 自分のデッキと対戦キャラクターの選択UI。
- `src/duelSetup.ts`: 選択キャラクターから対戦設定を解決する純粋関数。
- `src/game.ts`: beginner/challengerを含むゲーム生成とCPU行動選択。
- `src/App.tsx`: route、キャラクター読込、active snapshot、対戦開始、音声制御。
- `src/components/Overlays.tsx`: snapshot由来のカットイン表示。
- `src/savedDecks.ts`: 保存デッキの読込、検証、永続化。

## 6. UI設計

### キャラクター管理画面

- 左列: 組み込み・カスタム一覧、検索、新規追加。
- 右列: 基本情報、イラスト、セリフと音声の3タブ。
- 基本情報: 表示名、ID、使用デッキ、CPU難度、ゲーム内プレビュー。
- イラスト: 5種類のプレビュー、推奨比率、選択・差し替え・削除。
- セリフと音声: 15 cueのテキスト、音声選択、試聴、削除。
- フッター: 変更を戻す、削除、保存して反映。

画像の推奨比率は既存素材の実寸に合わせる。

- 通常・被弾・喜び: `1:1（正方形）`
- 切札・とどめ: `3:2（横長）`

音声は全セリフ共通で `1.5〜3秒程度` を推奨する。

### 対戦準備画面

- 自分のデッキ選択。
- 対戦キャラクターのカード一覧。
- 選択キャラクターのデッキ・CPU難度表示。
- キャラクター管理画面への導線。
- 設定変更は管理画面へ集約する。

## 7. エラー・互換方針

- 通常画像、表示名、IDを必須とする。
- IDは半角英小文字、数字、ハイフンのみとし、組み込み・保存済みIDとの重複を拒否する。
- 削除済み保存デッキなど未解決参照は具体的な理由を表示し、対戦開始を無効化する。
- IndexedDB操作は失敗時にも接続を必ず閉じる。
- 複数ファイルの非同期読込はfunctional state updateを使い、先に完了した変更を失わない。
- 保存デッキ一覧は壊れた1件だけを除外し、正常な保存デッキを維持する。
- 未知の将来versionのプロフィールstoreは上書きしない。
- `localStorage`書込失敗時はsession-onlyとして動作を継続し、永続化されていないことを表示する。
