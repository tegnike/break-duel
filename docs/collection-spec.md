# コイン・コレクション仕様

新弾（第2弾以降）のカード入手とパック開封に関する正本仕様。対戦ルールは `game-spec.md` を参照。

## コイン

| 項目 | 値 |
| --- | --- |
| 初期所持コイン | 1000 |
| 勝利報酬 | 10 コイン |
| 敗北・引き分け報酬 | 5 コイン |
| パック開封コスト | 5 コイン |
| 10連パック開封コスト | 50 コイン |

- 報酬は CPU 戦を最後まで打った試合（ライフ 0、リソース切れ、または最大手番の引き分け）にのみ付与する。
- 途中で「再戦」した試合とチュートリアルは報酬対象外。
- 残高は `localStorage`（キー: `break-duel:wallet`）に保存する。

## パック開封

- 1 パック 5 コイン・5 枚入り。中身は開封（封を剥いた）瞬間に抽選する。開封前の引き直しはできない。
- 開封前に単発または10連を選べる。10連は 50 コインで 10 パック（計50枚）を一括購入し、各パックを単発と同じ排出率で独立抽選する。パック内では同名カードを重複させず、別パック間の重複は許可する。
- 10連は最初の封を剥いた時点で全10パックの決済・抽選・コレクション保存を完了し、その後は購入済みの10パックを順番に開示する。各パックの5枚を公開し終えると「すべてめくる」が「次を開封する」に切り替わり、押すまでは公開済みカードを表示し続ける。10パック目の公開後は全50枚をパック単位でまとめた縦スクロール一覧を表示する。10連内で初めて入手したカードの `NEW` 表示は最初の1枚だけに付ける。
- 現行の暫定排出率: 1〜3 枚目 = N、4 枚目 = R、5 枚目 = R（60%）または SR（30%）または UR（10%）。
- レアリティは第 2 弾以降のカードだけが持つ。第 1 弾（スターター）カードにはレアリティを表示しない。
- 第 2 弾以降のレアリティは暫定的にカード属性から導出する: power 1 召喚獣・術式 = N、power 2 召喚獣・遺物 = R、power 3 召喚獣 = SR、power 4 召喚獣 = UR。
- 高レア（SR / UR）入りのパックは、開封の瞬間に確定演出（光柱）が発生する。

## 演出（ティア・カットイン）

- レアリティごとに演出色を分ける。SR = 金、UR = 虹色（hue-rotate で色相を回し続けるプリズム調）。
- カードをめくる挙動はレアリティや並び順（何枚目か）に関わらず全カード共通（即座にめくる）。SR/UR はめくった直後にステージ暗転・光条などの確定演出が入るが、めくる前の「タメ」演出は行わない。
- UR 確定でめくった瞬間は、`document.body` へ Portal した全画面カットイン（虹色の帯 + フラッシュ + 紙吹雪）を表示し、同時にステージ全体を軽く揺らすヒットストップ演出を入れる。
- 開封後のサマリーには第2弾コレクションの充実度（開封前→開封後の所持割合、進捗バー）を表示する。
- UR の虹色ホロ箔（低めの不透明度・回転する conic-gradient・screen 合成・キャライラスト部分は mask で除外）は `CardView` 共通コンポーネントに実装しており、パック開封に限らずカード一覧・デッキ制作・対戦画面（手札/場）など UR カードが表示される全ての場所に自動で適用される。

## コレクション

- 開封したカードは所持コレクション（キー: `break-duel:collection`、cardId → 所持枚数）に記録する。
- 第 1 弾（`Card.set` 未指定）のカードはスターター扱いで、各カードを 2 枚所持しているものとして表示・使用する。
- 第 2 弾以降（`Card.set >= 2`）のカードは、**所持枚数までしかデッキに入れられない**（デッキ保存時に検証。同名 2 枚の上限は従来通り併用）。
