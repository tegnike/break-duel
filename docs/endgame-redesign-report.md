## TL;DR

- **発端**: パスを覚えた最強 CPU（fair-gen005）同士を戦わせたら、**89.6% が 40 ターン引き分け**になった。
  最適プレイ同士だと誰も攻撃せず、試合が終わらない——ゲームのコアに「決着を強制する仕組みがない」
  という設計欠陥が露呈した
- **結果**: 5 ラウンドの比較検証を経て、**4 つのコアルール**（ライフ判定・手札上限 6・衰弱・
  手札防御 power3 以下）と**デッキ調整 3 件**を本採用。引き分け率は **89.6% → 0.3%**、
  全健全性指標が史上初めて同時に帯内へ入った
- **副産物**: プロジェクト全史で監視され続けた**先攻勝率 47% 問題が 49.1% で自然解消**。
  課題台帳（followups）は史上初の全クローズ
- **残り**: 中型スワーム（p2-3）だけは構造的強者として**監視項目**（60% 警報線）へ移行し、
  対抗手段は今後の**新カードで供給**する方針を明文化

## 1. 発端 — 最適プレイが暴いた「にらめっこ」

fair-gen005（ビーム探索 + パス比較 + 公開情報カウンティング）は、無駄な行動を一切しない
CPU になった。その結果、CPU 同士の対戦で:

- break vs control の **89.6% が 40 ターン上限の引き分け**（平均 38.5 ターン、中央値 = 上限）
- リード交代 13.7%、リーグの勝率表は draw 汚染で判読不能、ストレス回帰は長期化で完走不能

コード調査で構造要因を 3 つ特定した:

1. **攻撃は二重に期待値マイナス**: 見えない手札防御に刺さるリスク + ダメージを与えると
   その点数ぶん相手がドローする（ブレイク機構）
2. **待機が完全にタダ**: 手札上限なしで無限備蓄でき、手札が 1 枚でも残ると山札切れ決着が発動しない
3. **逃げ切り先に引き分け**: 40 ターン到達で無条件 draw

つまり「殴ると損、待つとタダ、逃げた先に引き分け」。これは CPU のバグではなく、
忍耐強い人間同士でも到達する**ゲーム側の均衡**だった。

## 2. 経緯 — 5 ラウンドの探索と 1 ラウンドの仕上げ

| 弾 | 試したこと | 結果と学び |
| --- | --- | --- |
| 第一弾 | 時計(C0+P1) / ブレイク緩和(P2) / シージ(P3) / 手札防御全廃(P4a) | 時計で膠着は完治（全構成 draw ≤1.2%）。P4a が質で別格 →「見えない後出し防御が病巣」が確定。ただし全廃は過剰（アグロ RISK・短期化）。P3 は廃案 |
| 第二弾 | 防御の中間用量: power≦2 制限(P4c) / セット防御 1 アクション(P4b) | 正反対に失敗。≦2 は安全弁が消えて全廃と同じ挙動、セットは「可視の防御 = 確実な抑止」で世界が凍り water が衰弱自滅。**支配変数は試合の長さ**という地図が完成 |
| 第三弾 | 診断ファースト + 内挿 2 点: power≦3(P4c3) / フリーセット(P4bf) | P4bf で切り分け完全決着（病巣は可視性でもコストでもなく「確実に機能する防御の存在」）。**P4c3 が歴代最接近**——スノーボールと先攻が初のゲート内。残るは p2-3 ストレスと単色帯のみ |
| 第四弾 | P4c3 仮採用 + 世界の再構築（デッキリバランス） | デッキ 3 件で**単色帯・先攻とも全回復**。ただし p2-3 だけはシーソー（break/control に回答を積むと自身が跳ねる）で、デッキ側の弾を撃ち尽くした |
| 第五弾 | 攻撃回数制限(A1) + ノブ 2 種 | **仮説反証**——スワームの本体はバーストでなく持続効率で、p2-3 は 58→56% と横ばい。しかも wind/先攻/beginner を壊した。全不採用 |
| 本採用 | A案: A1 なしで完成宣言、p2-3 は監視項目化 | 全ゲート合格。既定値化・UI・文書化・ガードレール再定義まで完了 |

## 3. 採用された変更（最終形）

### コアルール（4 件）

| ルール | 変更前 → 変更後 |
| --- | --- |
| ターン上限の決着 | 40 ターンで引き分け → **ライフの多い方の勝ち**（同値のみ引き分け） |
| 手札上限 | 無制限 → **6 枚**（自分のターン終了時に超過分をトラッシュ） |
| 衰弱 | なし → **自ターン開始時に山札 0 でドロー不能なら 1 ダメージ** |
| 手札防御 | 制限なし → **power 3 以下のカードのみ**（1 回/ターンは従来どおり） |

設計思想: **試合は必ず終わる**（衰弱 + ライフ判定）、**攻める理由がある**（power4 は手札で
止まらない）、**待つことにはコストがある**（手札上限 + 時計）。一方で「手札は逆転の資源」
（ブレイクドロー）は Break Duel の魂として無傷で残した。

### デッキ（3 件）

- fire: `AI-FIRE-4D → AI-FIRE-1B`（power4 過多の抑制）
- water: `CMD-WATER-RITE → CMD-TIDE-EDGE` / `AI-WATER-4 → AI-WATER-3D`（衰弱耐性。
  大量ドロー = 時計の前借り問題への適応）
- wind: `AI-WIND-4B → AI-WIND-3`

### CPU / 検証基盤

- challenger: **fair-gen005 凍結のまま無変更**（全 5 弾を同一計測器で比較）
- beginner: water のみ手札防御 power2 以下へ追従再較正（32% → 19.75%）
- `test:balance`: p2-3 閾値を 0.60 警報線へ再定義
- game-spec / design-principles: 新ルール正仕様化 + 時計世界の原則 + 全弾の廃案リスト収録
- UI: power4 手札防御の理由つき使用不可表示・手札上限トラッシュ演出・衰弱トースト・
  ライフ判定バナー・ルール画面/チュートリアル教材文の更新

## 5. 学び（前編 v1 レポートの教訓に追加するもの）

1. **最強の計測器は、ゲームの均衡そのものを暴く**。CPU が「上手くなる」最終段階は、
   ゲームの支配戦略（にらめっこ）を発見することだった。健全性の検証は「AI が下手を打たないか」
   ではなく「最適プレイ同士の世界が面白いか」を見る段階に到達した
2. **探索は括弧で挟んで内挿する**。全廃（効きすぎ）と現行（膠着）の両端を先に焼き、
   間を 1 段ずつ踏む——防御制限は「用量の階段」を 4 段試して 1 点に収束した
3. **切り分け実験は安い割に決定的**。フリーセット（P4bf）1 本で「病巣は可視性かコストか」の
   論争が永久に終わった
4. **ルール単体にリーグ帯まで要求しない**。時代遷移は毎回「ルール変更 → 世界が崩れる →
   リバランスで回収」の順。第四弾でこの原則を明文化してから収束が速くなった
5. **最後の 1 指標は「直す」以外の解もある**。p2-3 はルールの弾を撃ち尽くした末に
   「監視 + カードで解く」へ再定義した。コアルールを歪め続けるより、メタの回答を
   コンテンツとして供給するのが TCG の自然な進化
6. **診断ファーストは徹底すると効く**。第二弾（診断省略）は敗因特定が後手に回り、
   第三弾以降（診断 + 事前予測必須）は全ラウンドで予測が的中した
