# Break Duel 設計原則と検証基準

最終更新: 2026-07-09

この文書は、バランスやルールに触れる前に必ず読む「守るべき設計原則」と「検証の合格基準」をまとめた恒久ドキュメントです。元は `docs/archive/work-packages.md` 1.2 / 1.3 節にあった内容を、作業パッケージ完了に伴いここへ移しました。採用判断の根拠数値は `docs/balance-history.md`、現行ルールの正仕様は `docs/game-spec.md` を参照します。

## 1. 守るべき設計原則（変更前に必ず読むこと）

以下は検証の結果として意図的に選ばれた設計です。変更する場合はシミュレーションで再検証してください。

- **コスト = power の完全一致。** power 4 だけコストを変える案（3 で素出し）は「直感的でない」ため却下済み。基本 3 アクション化がその代替です。
- **試合は必ず終わる時計世界。** ターン開始ドロー不能時の衰弱 1 ダメージと、40 手番到達時のライフ判定を組み合わせる。ライフ同値だけは引き分けとして残すが、待つだけで結果を先延ばしにはできない。手札上限 6 も無限備蓄を防ぎ、時計を止めないための一部である。
- **手札は逆転の資源、盤面は優勢側の資源。** ブレイクドローで劣勢側の手札が増えるため、手札から撃つ除去（追撃粛清）は逆転装置として、盤面から撃つ除去（モンスター攻撃）は優勢側の決着装置として機能します。両者は互いの欠点（消耗戦化 / 雪だるま化）を相殺するため同時採用しています。
  - 時計世界でもブレイクドローは `ダメージ点数ぶん` の point 方式を維持する。手札上限 6 と power 3 以下の手札防御制限は待機・大型防御を抑えるが、被弾側へ反撃資源を渡す原則は変えない。
  - 補足（2026-07-05）: ルールの一貫性（本体への攻撃と同じ防御手段）を理由に、モンスター攻撃への手札防御割り込みを採用しました。手札防御には攻撃値以上の防御値を持つ手札と 1 ターン 1 回の共通枠が必要なため、決着装置としての機能は実測上ほぼ不変です（CPU 同士では 984 ストライク中 2 回しか発火せず、勝率・逆転率・ワンサイド率は変化なし。影響上限側の eager 運用でもリーグ PASS）。手札を盤面保護に使うと逆転資源が痩せるため、この原則自体は維持されています。検証数値は `docs/balance-history.md` の 2026-07-05 採用エントリを参照。
  - 補足（2026-07-07）: 同じ直感の延長として、モンスター攻撃への場防御（対象以外の未消耗召喚獣でかばう）も採用しました。盤面から盤面を守れるため影響は手札防御より大きく、`echoes` 強化と `fire` 弱体化が出ていますが、ワンサイド率は微減し、プレイフィール改善を優先して採用します。検証数値は `docs/balance-history.md` の 2026-07-07「モンスター攻撃への場防御追加」を参照。
- **デッキ切れは少量の勝ち筋として、即時敗北ではなく衰弱経由で残す。** 山札が尽きても場・手札が残る限り即敗北せず、必要なターン開始ドローに失敗するたび 1 ダメージを受ける。コントロールの消耗勝ちは維持しつつ、終盤に残った盤面とライフで最後の応酬を行える形を正とする。
- **中型スワーム（p2-3）は時計世界の構造的強者。** 低すぎない打点を複数並べ、衰弱・ライフ判定より先に詰めるため、コアルールだけで完全に 50% 以下へ押し込むことは他アーキタイプを壊す。p2-3 は合格ゲートではなく 60% 警報線つき監視項目とし、対抗手段は今後のカード追加で供給する。スワーム回答カードをカード追加の最優先テーマとする。

### 検証済みで却下された案（再提案しないこと。再提案するなら新データを添えること）

- 無条件除去の術式 → 逆転率が改善せず試合が延びるだけ（検証済み）
- 「消耗中の召喚獣への攻撃は必ず破壊」ルール → 消耗戦化のみでリターンなし（検証済み）
- デッキ 30 枚 → デッキ切れは 2% まで減るがキーカード希釈で逆転率が 44% まで低下（検証済み）
- power 3+ 上限の 6 枚以上への緩和 → 大型偏重で消化試合度が悪化（検証済み）。2026-07-06 に上限 1〜19 枚全域で再検証し、6 枚で即 RISK（総合勝率 54%）、以降 10 枚あたりまで頭打ちなく単調増加（14 枚以降で約 95% に飽和）と確認。5 枚未満（4 枚以下）も総合勝率 32.7% まで落ちて大型偏重戦略が非力になりすぎるため、現行の 5 枚が両側の失敗点に挟まれた最適点（`docs/balance-history.md` の 2026-07-06 エントリ参照）
- 序盤限定の召喚コスト割引 → 変則ルールとして却下（3 アクション化で代替）

### 終盤設計改訂・全 5 弾の廃案（再提案禁止）

2026-07-09 の終盤設計改訂では、C0 + P1 + P4c3 と第四弾デッキ変更 3 件だけを採用した。次の系統は比較計測で不採用になったため、新しいデータと明確に異なる問題設定がない限り再提案しない。

| 系統 | 廃案 | 不採用理由 |
| --- | --- | --- |
| 第一弾 | ブレイクドローを点数単位からイベント単位へ変える P2a、被弾側アクション補償 P2b/P2c | 逆転資源の原則を弱めるか、他指標を壊して完成ゲートを満たさなかった |
| 第一弾 | シージ / 戦線圧力 P3 | 試合を短くしすぎ、スノーボールを強めた |
| 第一〜二弾 | 手札防御全廃 P4a、power 2 以下だけにする P4c | draw は減るが短期化・スノーボール・p2-3 上振れが過大 |
| 第二〜三弾 | セット防御 P4b、無償セット P4bf、セット関連ノブ | draw・長期化・water 崩壊を同時に解けなかった |
| 第三弾 | 手札防御回数を試合全体で制限する案 | p2-3 への安全弁をさらに減らすため、失敗軸を悪化させると判断 |
| 第四弾 | p2-3 をコアルール不変のまま既存 6 デッキの差し替えだけで 50% 以下へ落とす案 | 単色帯とのシーソーが発生し、B1〜B7 の全試行で同時達成できなかった |
| 第五弾 | 攻撃 2 回 / 3 回制限、STRIKE を数える攻撃回数制限 | p2-3 を 50% 以下へ落とせず、wind・先攻率・beginner water に副作用 |

A1 `attacksPerTurnLimit`、セット防御系、ブレイクレート変更系などの実験フラグは互換・記録用に既定 off で残っていても、標準ルールではない。

## 2. デッキ構築ルール

25 枚 / 同名 2 枚まで / power 3 以上の召喚獣は合計 5 枚まで。

## 3. 検証基盤の使い方と合格基準

- リーグ検証: `npm run sim -- league --games-per-pair 100 --seed <seed> --decks break control fire water wind earth --out tmp/<name>`
  - 必ず 2 シード以上で回して平均を取ること（1 シードだと構成変更の影響とノイズを区別できない）
  - 合格基準: 単色 4 デッキが 45-55%、先攻勝率 48-52%
- 盛り上がり指標: `simulate` の `matches.jsonl` に各手番の `turn_start` イベントとライフ配列が記録されます。ここから「先制ダメージ手番」「リード交代回数」「勝者の最大ビハインド」「最終ライフ差」「決着形態」を集計できます。目安値は `docs/balance-history.md` の 2026-07-03 エントリを参照。
- カード別統計: `simulate` の `summary.json` 内 `card_usage` にカード別の使用・攻撃・防御などのイベント数が集計されます。
- バランス回帰: `npm run balance:cost -- --games-per-order 500 --seed <seed> --out tmp/<name>.json` でストレスデッキと既存 6 デッキの回帰確認ができます。GitHub Actions の `Balance Regression` ワークフローが週次（土曜朝 JST）と手動実行で同じチェックを回します。
  - p2-3 は 50% 以下の合格条件ではなく監視項目。60% 未満を正常域とし、60% を越えたら警報として原因を調査する。
- 全体チェック: `npm run check`（typecheck + vitest + build）。カード効果を追加・変更したら `src/game/cardEffectCoverage.test.ts` への登録が必須（怠ると unit テストが落ちる設計）。GitHub Actions の `CI` ワークフローが push / PR ごとに同じチェックを回します。
- 実装は TypeScript の単一実装です（`src/game.ts` / `src/game/actions.ts` が正。ブラウザ UI とシミュレーション CLI が同じエンジンを共有）。旧 Python シミュレータは 2026-07-08 に廃止しました。
- 終盤設計本採用後は時計世界の計測値が新基準。fair-gen005 時代以前のリーグ・盛り上がり・ストレス値と直接比較しない。
