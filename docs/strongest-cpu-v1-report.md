## TL;DR

- **目標**: バランス検証の計測器である challenger CPU を、ブラウザ即応・シード再現・公開情報のみという制約下で最強化する
- **結果**: 6 つの計画を経て **fair-gen004**（ビーム幅 5 のターン内プランニング + 公開情報カードカウンティング + 再調整済み重み）が完成。初代 CPU に対し世代を重ねるごとに明確に強くなり、最終世代は前世代に **6 割超**で勝ち越す
- **最大の副産物**: 途中で「出荷 CPU が相手の実手札を覗いていた」ことが発覚し、**公平化**した。さらに強くなった CPU が**ゲーム側の壊れ**（デッキ不均衡・高 power カードの壊れ）を 2 度暴き、カード/ルールごと健全化された
- **残り**: 監視 4 項目 + 実プレイで発見された悪手 1 件（下記 §6）

## 1. 目標と制約

challenger CPU はリーグ・盛り上がり指標・ストレスデッキ回帰・apex 探索すべての「計測器」。
計測器が弱いと、壊れカードの**見逃し**と、AI の下手さ由来の**誤検出**が起きる。
強化は今後のカード追加とルール調整の信頼性への基盤投資である。

| 制約 | 内容 |
| --- | --- |
| 思考時間 | ブラウザで 1 ターン 1 秒未満（最終実測: 平均 0.2ms / 最大 2.0ms） |
| 決定性 | 同一シードで再現可能（A/B 検証とビット一致確認の前提） |
| 公平性 | **公開情報のみ**参照（第 2 計画で追加。ガードテストで機械的に強制） |

## 2. 進め方（方法論）

全工程を「**計画書駆動**」で実施した。1 計画 = 1 ドキュメントで、§チェックリストが進行の正。
実装は別セッションが計画書だけを頼りに実行し、結果を同じドキュメントに記録する。

- **採用ゲート**: 対チャンピオンプール直接対決 55% 以上（2 シード一貫）+ デッキ床値非悪化
- **計測基盤**: `npm run gauntlet:ai`（ミラーデッキ総当たり）+ 歴代チャンピオンプール（メタ過学習防止）
- **安全装置**: ガードテスト（公開情報が同一なら行動も同一）、ビット一致確認、チュートリアル回帰
- **記録**: 採用も却下も `docs/balance-history.md` と各計画書に残す（再提案防止）
- **進化した運用**: 初期は「1 ゲート失敗で終了」の直列型 → 第 3 次から「失敗しても全トラック消化」の総力戦型に変更し、1 タームの成果密度を最大化

## 3. 経緯 — 6 幕の物語

| 幕 | 計画 | やったこと | 結果と発見 |
| --- | --- | --- | --- |
| 1 | 第 1 次（strongest-cpu-plan） | ガントレット基盤構築 + 重み探索 + 評価関数拡張 | 強化候補は全滅（50% 前後）。**計測基盤のみ採用**。「線形+貪欲はほぼ局所最適」の証拠が集まり始める |
| 2 | 公平化（fair-cpu-plan） | 結果分析中に**出荷 CPU が相手の実手札を参照**していることが発覚。ユーザー決定（B案）で公平化 | 覗き見 4 経路を除去し **fair-gen001** 凍結。旧版に 43.9% まで弱体化するが「人間と同条件」の正直な計測器を獲得 |
| 3 | リバランス第 1 次（fair-rebalance-plan） | 公平化で露見した**覗き見が隠していたバランス崩れ**（control 71.6% / water 35.2%）をカード側で修正 | 全単色 45-55% 帯へ回復。「強い計測器がゲームの真実を暴く」1 周目 |
| 4 | 第 2 次（strongest-cpu2-plan） | beginner に earth ミラーで負ける欠陥を診断 | challenger 側の修正は全滅、**beginner 較正のみ採用**（fair-gen002 = 重み同一）。「長期戦で資源を焼き尽くす」構造的弱点を発見 |
| 5 | 第 3 次・総力戦（strongest-cpu3-plan） | 過去 2 回自滅したビーム探索の**根本原因調査**を初実施 | **バグ発見**（盤面評価の多重計上 + 深い系列優先のタイブレーク）。修正後 beam3 が**プロジェクト初の 55% 突破（69%）**→ **fair-gen003**。副作用でリーグ・ストレスデッキが再崩壊（2 周目の暴き） |
| 6 | 第 4 次（strongest-cpu4-plan） | 強化を撃ち切ってから世界を 1 回で直す 2 部構成 | ビーム幅 5 + 重み再探索の**合成**で **fair-gen004**（対 gen003 60.9%）。積年の「相手ターン読み」は実装検証の上**正式却下**。power3/4 消耗登場のルール変更等で世界を健全化し、**v1 宣言** |

## 4. 最終成果物

### CPU: fair-gen004（`docs/assets/ai-champions/fair/fair-gen004.json`）

- ターン内プランニング: ビームサーチ幅 5（ターン終了盤面の終端評価・浅い系列優先）
- 公開情報カードカウンティング: `knownHandCards`（公開後に手札へ戻ったカードの追跡）+ 公開デッキリストベースの手札防御推定
- 重み: プランナー時代の地形で再探索した合成値（`damage=129` 等）
- 思考時間: 平均 0.2ms / 最大 2.0ms（1 秒制約の 1/500）

## 5. 学び（今後に効く教訓）

1. **計測器を強くするたびに、世界の壊れが新しく見える**。覗き見除去とプランナー導入の 2 回、リーグが「崩れて見えた」が、いずれも「弱い AI が隠していた真実」だった。CPU 強化とゲーム修正はセットの往復運動になる
2. **「失敗」の記録が後で勝ちに変わる**。第 2 次で落ちた 54.8% 候補、第 3 次で発明した合成テスト、「原因調査を前提条件にする」縛り——どれも過去の敗北の記録から生まれ、最終的な突破（beam バグ発見・gen004 合成）に直結した
3. **限界とバグを混同しない**。ビーム探索は 2 回「弱い」と判定されたが、真実は実装バグだった。性能が理不尽に悪いときは、チューニングの前に原因調査
4. **原則は機械で守る**。公平性は口約束ではなくガードテスト（公開情報同一 → 行動同一）で強制。以後どんな変更もこの網を通る
5. **CPU 同士の計測は万能ではない**。エンジン改善は両陣営に効いてガントレットで測れない。そして人間のプレイテストは、数千戦のシミュレーションが見逃した悪手を 1 戦で見つけた（§6）
